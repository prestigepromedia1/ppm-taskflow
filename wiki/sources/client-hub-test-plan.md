# Source: Client Hub Test Plan

Test plan for [[entities/taskflow|PPM TaskFlow]], generated by eng review on 2026-03-24.

## Test Infrastructure
- Backend: Jest (Worklenz default)
- Frontend: Vitest (Worklenz default)
- E2E: Playwright (to be added)
- Database: Test PostgreSQL instance with seed data

## Portal Routes Under Test
- `/portal/login` — magic link or password login
- `/portal/deliverables` — filtered task list (client-visible states only)
- `/portal/deliverables/:id` — single deliverable with approve/reject/comment
- `/portal/files` — file attachments scoped to client

## Key Interactions
- Magic link login → sees only their client's deliverables
- Task moved to `client_review` → appears in client portal
- Client approves → internal + master updated
- Client rejects → [[concepts/revision-loop|revision task]] created with structured feedback
- Client comments → action item created internally
- Retainer rollup shows correct hours/budget %

## Edge Cases
- Client accessing internal project → 403
- Deactivated client login → rejected
- Task synced when no client project linked → graceful no-op
- Retainer period rollover
- Concurrent approval by two client users
- Expired magic link → clear error + resend

## Critical E2E Paths
1. Full approval flow: create → client_review → approve → all layers updated
2. Full rejection flow: reject → revision → rework → resubmit → review
3. Portal access control: client can ONLY see client-layer projects

## Worklenz Regression Tests
- Task CRUD after schema changes
- `create_task` function handles projects with `board_layer`
- Project creation unaffected
- Kanban drag-and-drop (Socket.IO)
- Time tracking entries still record correctly
- Auth flow (login, signup, Google OAuth, sessions)
- Notifications (in-app + email)
- File upload/download
